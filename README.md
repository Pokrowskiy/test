## Сервер :

init - создаём рабочую директорию, если её нет и запускаем цикл, который бесконечно пытается подключиться к rabbitmq, пока не получится, либо его не прервут.

add_message - генерируем уникальный ID изображения через UUID, после чего отправляем сообщение с использованием pika.BasicProperties(delivery_mode=2) (режим 2 позволяет сообщениям сохраняться в случае перезагрузки сервера, что повышает надёжность)

get_processed_images - проверяет рабочую директорию, собирает имена всех файлов. Тут может быть не очень надёжный момент, так как нам не гарантируется, что в рабочей директории содержатся только файлы - изображения.

get_image_caption - проверяет существование файла с нужным id, если есть - возвращает его содержимое, если нету - None

## Воркер

init - создаём рабочую директорию, если её нет и запускаем цикл, который бесконечно пытается подключиться к rabbitmq, пока не получится, либо его не прервут. basic_qos(prefetch_count=1) - ограничиваем количество сообщений, которые можно отправить без подтверждения

callback - сохраняет изображения в рабочую директорию, но обратная связь у меня не очень работает.

run - в основном поддерживает соединение на случай, если оно вдруг прервётся. Крутит подключение в цикле, пока не будет завершено

ну и в мейне запускается сам воркер.
